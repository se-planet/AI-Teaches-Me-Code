## ✅ 一、数组（Array）

> 📅 学习时间：2026-07-01 ~ 2026-07-02
> 📂 代码：[01_数组/lesson_array.py](01_数组/lesson_array.py)

### 1.1 核心概念

| 特性 | 说明 |
|------|------|
| **存储方式** | 一块连续的内存空间，像一排紧挨的停车位 |
| **索引访问 O(1)** | 知道首地址 + 偏移量，直接算出任意元素位置 |
| **插入/删除 O(n)** | 中间插入要挪动后面所有元素 |
| **查找 O(n)** | 最坏情况要遍历整个数组 |

### 1.2 时间复杂度速查

| 操作 | 复杂度 | 原因 |
|------|:--:|------|
| 按索引读 | O(1) | 连续内存，直接计算 |
| 按索引写 | O(1) | 同上 |
| 尾部插入 `append` | O(1) | 末尾有空位 |
| 中间插入 `insert` | O(n) | 后面元素要后移 |
| 按值查找 | O(n) | 一个个遍历 |
| 删除 | O(n) | 后面元素要前移 |

### 1.3 核心技巧：双指针

两个指针从不同位置出发，向中间（或同向）移动。

**常见变体：**
- **左右指针**：反转数组
- **快慢指针**：去重、移动零

**模板：**
```python
left, right = 0, len(arr) - 1
while left < right:
    arr[left], arr[right] = arr[right], arr[left]  # 交换
    left += 1
    right -= 1
```

### 1.4 两数之和（Two Sum）详解

> LeetCode 第 1 题，新手必刷

**暴力法 O(n²)：** 每两个数都加一遍试试。

```python
for i in range(len(nums)):
    for j in range(i + 1, len(nums)):
        if nums[i] + nums[j] == target:
            return [i, j]
```

**哈希法 O(n)：** 把加法变成减法，算搭档，查字典。

```python
seen = {}
for i, num in enumerate(nums):
    complement = target - num
    if complement in seen:
        return [seen[complement], i]
    seen[num] = i
```

**关键：先查后存。** 先 `seen[num]=i` 再查会导致 `[3,3]` target=6 找到自己。

| | 暴力 | 哈希 |
|------|------|------|
| 思路 | 两两相加试 | 算搭档，翻笔记本 |
| 时间 | O(n²) | O(n) |
| 空间 | O(1) | O(n) |

### 1.6 Python 注意点

- `list.sort()` 返回 `None`（原地排序），`sorted(list)` 返回新列表
- 切片是左闭右开：`arr[a:b]` 含 a 不含 b
- `arr2 = arr1` 是浅拷贝 → 用 `arr1[:]` 或 `arr1.copy()`

## ✅ 二、链表（Linked List）

> 📅 学习时间：2026-07-03
> 📂 代码：[02_链表/lesson_linked_list.py](02_链表/lesson_linked_list.py)

### 2.1 链表 vs 数组

| | 数组 | 链表 |
|------|------|------|
| 内存 | 连续一块 | 散落各处，指针相连 |
| 索引访问 | O(1) 瞬间 | O(n) 必须从头走 |
| 插入/删除 | O(n) 挪位置 | O(1) 改指针就行（头部） |
| 查找 | O(n) | O(n) |
| 比喻 | 停车场一排紧挨车位 | 火车，车厢用钩子相连 |

### 2.2 节点结构

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val      # 数据
        self.next = next    # 指向下一个节点的指针
```

### 2.3 基本操作

| 操作 | 复杂度 | 关键点 |
|------|:--:|------|
| 头部插入 | O(1) | 新节点 → 旧头 |
| 尾部插入 | O(n) | 走到末尾再挂 |
| 删除 | O(n) | 先找到，跳过它 |
| 遍历 | O(n) | `while cur:` |

### 2.4 核心技巧

**反转链表（四步公式）：**

```python
prev = None; cur = head
while cur:
    nxt = cur.next      # ① 记后路
    cur.next = prev     # ② 箭头调头（核心）
    prev = cur          # ③ prev 前进
    cur = nxt           # ④ cur 前进
return prev
```

**快慢指针：**

| 应用 | 技巧 |
|------|------|
| 找中点 | 快走两步、慢走一步 |
| 判环 | 快慢指针相遇 = 有环 |

### 2.5 关键原则

- **head 不动，cur 走路** — head 是门牌号，丢了就找不到链表了
- **改指针前先记后路** — `nxt = cur.next` 后再改 `cur.next`
- **空链表先判断** — `if not head: return`
- **dummy 看门，cur 干活** — dummy 守在起点不动

|:--:|------|------|:--:|
| 206 | 反转链表 | 迭代 + 递归（双解 AC） | ✅ |
| 141 | 环形链表 | 快慢指针 | ✅ |

### 2.7 递归反转链表（进阶）

```python
def reverseList(self, head):
    if not head or not head.next:
        return head
    new_head = self.reverseList(head.next)
    head.next.next = head
    head.next = None
    return new_head
```

### 2.8 Python 注意点

- `cur.next` 读是取值，`cur.next = xxx` 是改箭头
- `while cur` 遍历每个节点，`while cur.next` 停在最后一个
- LeetCode 上 ListNode 已定义好，本地需自己定义

## ✅ 三、栈与队列（Stack & Queue）

> 📅 学习时间：2026-07-09
> 📂 代码：[03_栈与队列/lesson_stack_queue.py](03_栈与队列/lesson_stack_queue.py)

### 3.1 核心概念

| | 栈（Stack） | 队列（Queue） |
|------|------------|-------------|
| 规则 | LIFO 后进先出 | FIFO 先进先出 |
| 插入 | `append()` O(1) | `append()` O(1) |
| 删除 | `pop()` O(1) | `popleft()` O(1) |
| 看顶端 | `stack[-1]` | `queue[0]` |
| 比喻 | 一摞盘子，后放先拿 | 排队买票，先来先走 |
| Python | `list` | `collections.deque` |

### 3.2 关键要点

- **栈用 `list`**：`append()` + `pop()` 都是 O(1)
- **队列用 `deque`**：`popleft()` 是 O(1)，`list.pop(0)` 是 O(n) ❌
- `deque` 底层是双向链表，头部删除不用挪

### 3.3 经典应用

**括号匹配（LeetCode 20）：**

```python
def isValid(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in pairs:
            if not stack or stack.pop() != pairs[ch]:
                return False
        else:
            stack.append(ch)
    return not stack
```

**最小栈（LeetCode 155）：** 两个栈同步维护，一个存数据，一个存当前最小值。

**双栈模拟队列（LeetCode 232）：** 两个栈来回倒，反转两次恢复原序。

## ✅ 五、二叉树（Binary Tree）

> 📅 学习时间：2026-07-13
> 📂 代码：[05_树/lesson_tree.py](05_树/lesson_tree.py)
> 🧠 本章核心：学会递归思维

### 5.1 核心概念

```
         1         ← 根（root）
        / \
       2   3       ← 左孩子、右孩子
      / \
     4   5         ← 叶子（没有孩子的节点）
```

| 术语 | 含义 |
|------|------|
| 根 root | 最顶层的节点 |
| 叶子 leaf | 没有孩子的节点 |
| 子树 subtree | 任何节点 + 它下面所有后代 |
| 深度 depth | 从根到最远叶子经过的节点数 |

### 5.2 三种遍历（核心！）

```
       1
      / \
     2   3
    / \
   4   5

前序（根左右）：1 2 4 5 3    — 先打印自己
中序（左根右）：4 2 5 1 3    — BST 中序 = 排序结果
后序（左右根）：4 5 2 3 1    — 先处理孩子，最后处理自己
```

### 5.3 递归模板（所有树题通用）

```python
def solve(root):
    if not root:          # 空节点 → 退出
        return
    left = solve(root.left)    # 处理左子树
    right = solve(root.right)  # 处理右子树
    return 合并(root, left, right)
```

### 5.4 二叉搜索树（BST）

> 左 < 根 < 右，对所有节点成立。查找 O(log n)。

### 5.5 LeetCode 实战

| 题号 | 题目 | 技巧 | 状态 |
|:--:|------|------|:--:|
| 104 | 二叉树的最大深度 | 递归 | ✅ |
| 100 | 相同的树 | 双树同步递归 | ✅ |
| 226 | 翻转二叉树 | 交换左右 + 递归 | ✅ |

### 5.6 Python 注意点

- 树节点是对象，`.left` / `.right` 是指针，`.val` 是数据
- `self.xxx()` 是调类的方法（递归），跟 `self.root`（不存在）不是一回事
- 递归先写终止条件 `if not root`
- 递归返回值就是普通值，别再对返回值递归

## ✅ 六、堆（Heap）

> 📅 学习时间：2026-07-13
> 📂 代码：[06_堆/lesson_heap.py](06_堆/lesson_heap.py)

### 6.1 核心概念

> 最小堆：每个父 <= 子，堆顶是全堆最小值
> 最大堆：每个父 >= 子，堆顶全堆最大值

Python 只有最小堆，最大堆用负数实现。

### 6.2 核心操作

```python
import heapq

heapq.heapify(arr)          # 建堆 O(n)
heapq.heappush(heap, x)     # 插入 O(log n)
heapq.heappop(heap)         # 弹出最小 O(log n)
heap[0]                     # 看堆顶 O(1)
```

### 6.3 Top K 模板

```python
# 找第 K 大：维护大小为 K 的最小堆
heap = []
for num in nums:
    heapq.heappush(heap, num)
    if len(heap) > k:
        heapq.heappop(heap)
return heap[0]   # 堆顶 = 第 K 大
```

### 6.4 LeetCode 实战

| 题号 | 题目 | 技巧 | 状态 |
|:--:|------|------|:--:|
| 215 | 数组中的第K个最大元素 | 大小为k的最小堆 | ✅ |

### 6.5 Python 注意点

- 最大堆技巧：存负数 `-x`
- `heap[0]` 看堆顶，`heappop` 弹出堆顶
- 堆不保证全序，只保证 `heap[0]` 最小
- `range()` 参数要整数，不能传列表

## ✅ 七、图（Graph）

> 📅 学习时间：2026-07-14
> 📂 代码：[07_图/lesson_graph.py](07_图/lesson_graph.py)

### 7.1 核心概念

> 图 = 节点（顶点）+ 边。邻接表 = `{节点: [邻居列表]}`。

### 7.2 BFS vs DFS

| | BFS | DFS |
|------|------|------|
| 全称 | Breadth-First Search | Depth-First Search |
| 数据结构 | 队列（deque） | 递归 |
| 方式 | 一层层扩散 | 一条路走到底 |
| 比喻 | 水波扩散 | 钻迷宫 |
| 找最短路径 | ✅ | ❌ |

### 7.3 网格淹没法模板

```python
def dfs(r, c):
    if 越界 or 不是目标: return
    grid[r][c] = 标记    # 淹掉/染色
    dfs(r+1, c); dfs(r-1, c)   # 上下
    dfs(r, c+1); dfs(r, c-1)   # 左右
```

### 7.4 岛屿 vs 图像渲染

| | 岛屿数量 | 图像渲染 |
|------|----------|----------|
| 起点 | 未知，全图扫描 | 给定 (sr, sc) |
| 条件 | `== '1'` | `!= old_color` |
| 处理 | 淹成 `'0'` | 染成 `color` |
| 计数 | 需要 count | 不需要 |
| for 循环 | 需要 | 不需要（从起点 DFS） |

### 7.5 LeetCode 实战

| 题号 | 题目 | 技巧 | 状态 |
|:--:|------|------|:--:|
| 200 | 岛屿数量 | DFS 淹没法 | ✅ |
| 733 | 图像渲染 | DFS 染色 | ✅ |

### 7.6 Python 注意点

- 函数调用用 `()`，别用 `[]`
- `r >= rows, c >= cols`（索引从 0 开始，等号不能漏）
- DFS 条件用 `!= target` 不是 `== target`
- `image[r][c] = color` 赋值，不是 `== color`

## 🔥 八、排序算法

> 📅 学习时间：2026-07-14 ~ 2026-07-16
> 📂 代码：[08_排序/lesson_sort.py](08_排序/lesson_sort.py)

### 8.1 排序全家福

| 算法 | 时间 | 空间 | 一句口诀 |
|------|:--:|:--:|------|
| 冒泡 | O(n²) | O(1) | 大的往后冒 |
| 选择 | O(n²) | O(1) | 挑最小的放前面 |
| 插入 | O(n²) | O(1) | 摸牌，插到合适位置 |
| **快排** | **O(n log n)** | O(log n) | 分区 + 递归 |
| **归并** | **O(n log n)** | O(n) | 拆半 + 合并 |

### 8.2 重点：快排 & 归并

**快排：** 选 pivot，左小右大，递归排两边。
**归并：** 分到不能再分，两两有序合并。

### 8.3 Python 实战

```python
sorted(arr)     # 返回新列表
arr.sort()      # 原地排序，返回 None
```

### 8.4 原地快排 vs 归并

| | 原地快排 | 归并 |
|------|------|------|
| 写法 | swap 交换 | 建新数组合并 |
| 空间 | O(log n) | O(n) |
| LeetCode 912 | 极端数据会超时 | ✅ 100%稳过 |
| 面试 | 展示原地操作 | 展示分治合并 |

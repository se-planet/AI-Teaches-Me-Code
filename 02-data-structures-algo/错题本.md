## ❌ 1. `.sort` vs `.sort()`

**日期：** 2026-07-02 | **来源：** 练习题1「合并有序数组」

**错误代码：**
```python
return arr3.sort
```

**现象：** 返回了一个方法对象，不是排序后的列表。

**原因：**
- `.sort` 不加括号拿到的是方法本身，没执行
- 就算写成 `.sort()`，它返回的是 `None`（原地排序），不是排好的数组

**正确写法：**
```python
arr3.sort()  # 原地排序
return arr3   # 再返回数组
```

**教训：** Python 中 `.sort()` 返回 `None`，需要排序后返回新列表时用 `sorted(arr)`。

## ❌ 2. 切片左闭右开

**日期：** 2026-07-02 | **来源：** 练习题2「最大子数组和」

**错误代码：**
```python
sub = nums[start: end]   # 漏掉了 end 位置的元素
```

**现象：** `start=0, end=0` 时切出空数组 `[]`，导致 `sum([])` = `0`。

**原因：** Python 切片规则是 `[起始: 结束)`，含起始不含结束。

**正确写法：**
```python
sub = nums[start: end + 1]  # end 也要包含
```

**教训：** 取子数组时记得 +1，或直接 `range(start, len(nums))` 配合 `end+1`。

## ❌ 3. 暴力法内循环范围写错

**日期：** 2026-07-03 | **来源：** LeetCode 1「两数之和」暴力法

**错误代码：**
```python
for j in range(1, len(nums)):    # 永远从1开始
```

**现象：** `i=2` 时 `j` 又从 `1` 开始比，造成重复比较。

**原因：** `i` 前面的数已经被外层循环跟当前数比过了，不用再比。

**正确写法：**
```python
for j in range(i + 1, len(nums)):  # 从 i 后面开始
```

**教训：** 内循环起始值跟着外循环变量走，避免重复计算。

## ❌ 4. `index()` 没指定起始位置

**日期：** 2026-07-03 | **来源：** LeetCode 1「两数之和」哈希法尝试

**错误代码：**
```python
j = nums.index(see)    # 永远从0开始找
```

**现象：** `nums=[3, 3], target=6` 时，`index(3)` 永远返回 `0`，导致 `[0, 0]`。

**正确写法：**
```python
j = nums.index(see, i + 1)   # 从 i+1 开始找，跳过自己
```

**教训：** 用 `index()` 找元素时注意指定起始位置，避免找到自己。

## ⚠️ 5. `cur.next` 读 vs 写混淆

**日期：** 2026-07-03 | **来源：** 链表学习「反转链表」

**混淆点：** `cur.next` 在不同语境下含义不同。

- **读（取值）：** `nxt = cur.next` → 拿到下一节点
- **写（改箭头）：** `cur.next = prev` → 把箭头掰到别处

**类比：**
```python
x = 5    # 读：x 的值是 5
x = 10   # 写：把 10 塞进 x
```

**教训：** `=` 号右边的 `cur.next` 是取值，左边的 `cur.next` 是改箭头。不是一个意思。

## ⚠️ 6. `while cur` vs `while cur.next`

**日期：** 2026-07-03 | **来源：** 链表遍历 vs 找尾部

| 条件 | 停在哪里 | 用途 |
|------|----------|------|
| `while cur` | None（过了末尾） | 遍历每个节点 |
| `while cur.next` | 最后一个节点 | 找尾部准备插入 |

**常见错误：** 用 `while cur.next` 遍历 → 最后一个节点被跳过。

**教训：** 想处理每个节点用 `while cur`，想在末尾停下用 `while cur.next`。

## ❌ 7. 反转链表：`nxt = head.next` 写成定值

**日期：** 2026-07-03 | **来源：** LeetCode 206「反转链表」

**错误代码：**
```python
nxt = head.next    # head.next 永远是 ②
```

**现象：** 每轮保存的"后路"都是同一个节点 ②，cur 被困在 ② 死循环。

**正确写法：**
```python
nxt = cur.next     # 各节点的后路都不一样
```

**教训：** 反转公式里每一步都跟 `cur` 相关，不是 `head`。

## ❌ 8. 环形链表：慢指针也走了两步

**日期：** 2026-07-03 | **来源：** LeetCode 141「环形链表」

**错误代码：**
```python
slow = slow.next.next    # 慢指针跟快指针一样快
```

**现象：** 快慢差距永远不变，永远追不上。

**正确写法：**
```python
slow = slow.next         # 慢一步，快两步
```

**教训：** 快慢指针的核心是**步差=1**，每轮差距缩1，迟早追上。

## ❌ 9. 环形链表：`return False` 放在循环里面

**日期：** 2026-07-03 | **来源：** LeetCode 141「环形链表」

**错误代码：**
```python
if fast == slow:
    return True
else:
    return False    # 第一轮没碰上就退出
```

**现象：** 第一轮没相遇就直接返回 False，后面的都不查。

**正确写法：**
```python
if fast == slow:
    return True
# 循环外
return False        # 循环跑完才说"没环"
```

**教训：** 否定判断放在循环外，必须是"遍历完都没找到"才成立。

## ❌ 10. `list.append[ch]` — 方括号当圆括号

**日期：** 2026-07-09 | **来源：** 练习题1「有效的括号」

**错误代码：**
```python
stack.append[ch]     # 方括号！
```

**现象：** 语法错误，`append` 是方法，必须用圆括号调用。

**正确写法：**
```python
stack.append(ch)     # 圆括号
```

**教训：** 方法调用永远是 `fn(args)` 圆括号，`[]` 是索引取值用的。

## ❌ 11. 括号匹配：缺少左右判断

**日期：** 2026-07-09 | **来源：** 练习题1「有效的括号」

**错误代码：**
```python
for ch in s:
    if not stack or parts[ch] != stack.pop():
        return False    # 左括号也在 parts 里查 → KeyError！
```

**原因：** 没有先判断 `ch in parts`，左括号不在字典 key 里就崩了。

**逻辑：** 碰到字符先问——你是右括号吗？
- 是 → 弹栈匹配
- 不是 → 你是左括号，存进栈

**正确结构：**
```python
for ch in s:
    if ch in parts:         # 先判断是不是右括号
        if not stack or stack.pop() != parts[ch]:
            return False
    else:                    # 是左括号，存起来
        stack.append(ch)
```

**教训：** 括号匹配 = 左括号 push + 右括号 pop，必须区分对待。

## ❌ 12. 判重用 list 而不是 set

**日期：** 2026-07-10 | **来源：** 练习题1「存在重复元素」

**错误代码：**
```python
se = []                    # 用列表做判重
if i not in se:           # O(n) 查找！
    se.append(i)
```

**问题：** `i not in list` 每次都要遍历整个列表，O(n²) 总复杂度。

**正确写法：**
```python
seen = set()              # set O(1) 查找
if num in seen:
    return True
seen.add(num)
```

**教训：** 判重用 `set`，别用 `list`。O(1) vs O(n)。

## ❌ 13. True/False 返回值反了

**日期：** 2026-07-10 | **来源：** 练习题1「存在重复元素」

**错误代码：**
```python
if num in seen:
    return False          # 找到重复却说没重复
return True               # 走完没找到却说有重复
```

**教训：** 写之前先想清楚函数名问什么。`contains_duplicate` → 有重复吗 → 有就 True。

## ❌ 14. 第二个循环变量写错

**日期：** 2026-07-10 | **来源：** 练习题2「字母异位词」

**错误代码：**
```python
for i in t:
    count_t[i] = ...     # 变量名是 t，循环却跑 s
```

**现象：** `count_t` 也是 s 的统计，两个一样，永远 True。

**教训：** 复制粘贴后检查循环变量名。

## ❌ 15. 变量名和字符串名混淆

**日期：** 2026-07-10 | **来源：** LeetCode 242「字母异位词」

**错误代码：**
```python
count_s = {}
for i in s:
    self.s[i] = self.s.get(i, 0) + 1  # ❌ 字典名写成了字符串名
```

**问题：** `count_s` 是计数字典，`s` 是原字符串。且普通函数不用 `self`。

**正确写法：**
```python
count_s[i] = count_s.get(i, 0) + 1
```

**教训：** 计数器和数据源用不同前缀区分，如 `count_s` vs `s`。
